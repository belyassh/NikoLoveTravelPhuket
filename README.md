# Niko Travel

Интерактивная витрина экскурсий для туристического агентства: каталог, фильтры, SEO-страницы карточек, форма подбора на главной и форма бронирования на страницах экскурсий с отправкой на email и резервом через Telegram.

## Возможности

- Данные экскурсий полностью управляются из JSON.
- Автогенерация SEO-страниц категорий и карточек услуг из JSON перед dev/build.
- Поиск по названию, описанию и тегам.
- Фильтр по направлениям.
- Детальная карточка с фото-слайдером.
- Поддержка нескольких программ одной экскурсии с отдельной логикой бронирования на product page.
- Отправка заявки через email endpoint.
- Fallback в Telegram, если email-сервис недоступен.
- Передача UTM-меток и click id в лиды, плюс событие `generate_lead` для аналитики и рекламы.
- Адаптивная верстка для desktop и mobile.

## Демо-поток пользователя

1. Пользователь открывает каталог и выбирает экскурсию.
2. Переходит на SEO-страницу выбранной экскурсии.
3. Выбирает программу/отправление и дату экскурсии.
4. Отправляет бронирование на email, либо переходит в Telegram при fallback.

## Быстрый старт

1. Клонируйте репозиторий.
2. Установите зависимости.
3. Запустите dev-сервер.

```bash
npm install
npm run dev
```

После запуска откройте адрес из терминала (обычно http://localhost:5173).

## Production-сборка

Собрать оптимизированную версию:

```bash
npm run build
```

Локальный предпросмотр production-версии:

```bash
npm run preview
```

Готовые статические файлы будут в папке dist.

## Деплой на GitHub Pages

Проект настроен на автоматический деплой production-сборки через GitHub Actions.

Workflow:

- [ .github/workflows/deploy-pages.yml ](.github/workflows/deploy-pages.yml)

Как работает:

1. При каждом push в ветку main запускается npm ci и npm run build.
2. Папка dist публикуется в GitHub Pages.

Требуется один раз включить в настройках репозитория:

1. Settings → Pages.
2. В Source выбрать GitHub Actions.

## Альтернатива без сборки

Если нужен только простой статический запуск в текущем виде, можно использовать любой локальный сервер.

Пример через Python:

```bash
python3 -m http.server 8080
```

Адрес:

```text
http://localhost:8080
```

## Конфигурация

Основные настройки и контент находятся в JSON-файлах:

- data/excursions.json
- data/rentals.json
- data/services.json

### Основные поля

| Поле | Тип | Описание |
| --- | --- | --- |
| agency.name | string | Название агентства |
| agency.currency | string | Валюта цен (например, USD, EUR) |
| telegram.managerUsername | string | Username менеджера без @ |
| emailService.endpoint | string | Endpoint формы для отправки email |
| excursions | array | Список экскурсий |

### Поля объекта экскурсии

| Поле | Тип | Описание |
| --- | --- | --- |
| id | string | Уникальный идентификатор |
| title | string | Название экскурсии |
| overview | string | Краткое описание для карточки |
| description | string | Полное описание для модалки |
| price | number | Базовая или минимальная цена за 1 человека |
| programs | array | Варианты программы/отправления с отдельной ценой и временем выезда |
| included | string[] | Что включено |
| bring | string[] | Что взять с собой |
| tags | string[] | Категории для фильтра |
| images | string[] | Ссылки на изображения |

## Управление экскурсиями

1. Откройте data/excursions.json.
2. Добавьте, измените или удалите объект в массиве excursions.
3. Сохраните файл и обновите страницу.

## Отправка заявок

### Email

Укажите emailService.endpoint, например endpoint Formspree. При успешном ответе сервиса пользователь получает подтверждение на странице.

### Telegram fallback

Если email endpoint недоступен, приложение открывает Telegram как резервный канал связи. При наличии managerUsername ссылка ведет напрямую на чат менеджера.

### Атрибуция и реклама

Главная форма подбора и формы на SEO-страницах передают в лиды `landingPage`, `referrer`, UTM-параметры и click ids (`gclid`, `fbclid`, `yclid`, `msclkid`). На успешной отправке и на Telegram fallback уходит событие `generate_lead`, которое можно использовать в GA4 и Google Ads.

## Технические заметки

- Проект поддерживает как режим сборки через Vite, так и статический запуск без сборки.
- Для загрузки JSON нужен запуск через локальный сервер, а не прямое открытие файла.
- В логике предусмотрен fallback для отсутствующих изображений в карточках и модалке.
- Перед командами npm run dev и npm run build автоматически запускается генерация SEO-страниц (predev/prebuild).

## SEO перед запуском

Реализованный техминимум:

- robots: robots.txt
- sitemap: sitemap.xml
- favicon: image.png подключен в head
- 404-страница: 404.html
- Open Graph / Twitter метатеги и canonical в index.html
- JSON-LD (TravelAgency) в index.html

Перед подключением собственного домена обновите:

1. URL в canonical, og:url и og:image в index.html.
2. URL в robots.txt (строка Sitemap).
3. URL в sitemap.xml (поле loc).

После привязки домена в GitHub Pages:

1. Включите Enforce HTTPS.
2. Добавьте сайт в Google Search Console.
3. Отправьте sitemap.xml в Search Console.
